# מחולל מסמכי PDF עם לוגו 🌿

אתר קטן ופשוט: בוחרים לוגו (GARDENIT / המשתלה של בועז), מדביקים טקסט או הצעת מחיר,
ומקבלים קובץ **PDF** מעוצב להורדה או לשליחה (וואטסאפ / מייל).

האתר **סטטי לחלוטין** — רק קבצי HTML/CSS/JS, בלי שרת ובלי בסיס נתונים.
הכול קורה בדפדפן של המשתמש. לכן הוא חינמי לאחסון ומהיר מאוד.

---

## 📁 מבנה הקבצים

```
index.html                  – הדף הראשי (ממשק המשתמש)
style.css                   – העיצוב
app.js                      – הלוגיקה: תצוגה מקדימה + יצירת ה‑PDF
vendor/pdf-lib.min.js       – יצירת קובצי PDF
vendor/fontkit.umd.min.js   – הטמעת גופן TTF בתוך ה‑PDF
vendor/bidi.min.js          – סידור טקסט דו‑כיווני (עברית + מספרים)
font-data.js                – גופן Alef מוטמע (Regular + Bold, רישיון OFL)
logo-data.js                – הלוגואים מוטמעים כ‑data‑URI
logos/                      – קבצי הלוגו (לתצוגה ולתמונת ה‑PDF)
render.yaml                 – הגדרות פריסה ל‑Render (לא חובה)
הוראות-לאמא.md              – מדריך פשוט בעברית למשתמשת
```

## ▶️ הרצה מקומית (לבדיקה במחשב)

צריך פשוט לפתוח דרך שרת קטן (לא דאבל‑קליק על הקובץ, כדי שהתמונות ייטענו נכון):

```bash
# מתוך התיקייה של הפרויקט
npx serve .
# או, אם מותקן פייתון:
python -m http.server 8000
```

ואז לפתוח בדפדפן את הכתובת שמודפסת (לדוגמה http://localhost:3000 או http://localhost:8000).

## 🚀 העלאה לאוויר — GitHub + Render (שלב‑אחר‑שלב)

> צריך לעשות את זה **פעם אחת**. אחרי זה יש כתובת קבועה לשלוח לאמא.

### חלק א' — העלאה ל‑GitHub

1. נכנסים ל‑https://github.com ומתחברים (או נרשמים — חינם).
2. לוחצים על **+** למעלה מימין → **New repository**.
3. שם: למשל `gardenit-quotes`. משאירים **Public**. לא מסמנים שום תיבה נוספת.
   לוחצים **Create repository**.
4. בעמוד שנפתח יש פקודות. במחשב, בתוך תיקיית הפרויקט, מריצים בטרמינל:

   ```bash
   git init
   git add .
   git commit -m "אפליקציית PDF עם לוגו"
   git branch -M main
   git remote add origin https://github.com/USERNAME/gardenit-quotes.git
   git push -u origin main
   ```

   (מחליפים `USERNAME` בשם המשתמש שלכם ב‑GitHub.)
   אם מבקשים סיסמה — צריך **Personal Access Token** (Settings → Developer settings →
   Tokens), לא הסיסמה הרגילה.

   > הערה: ב‑repo הזה כבר הרצנו `git init` ויצרנו commit ראשון, אז אפשר לדלג
   > ישר ל‑`git remote add ...` ו‑`git push`.

### חלק ב' — פריסה ל‑Render

1. נכנסים ל‑https://render.com ומתחברים (אפשר עם חשבון GitHub — הכי קל).
2. לוחצים **New +** → **Static Site**.
3. בוחרים את ה‑repository `gardenit-quotes` (אם מבקשים — מאשרים ל‑Render גישה ל‑GitHub).
4. ממלאים:
   - **Name:** `gardenit-quotes` (זה ייכנס לכתובת).
   - **Build Command:** משאירים **ריק**.
   - **Publish Directory:** כותבים נקודה `.` (או משאירים ריק).
5. לוחצים **Create Static Site**. תוך כדקה האתר עולה.
6. מקבלים כתובת קבועה כמו:
   **https://gardenit-quotes.onrender.com**
   זו הכתובת לשלוח לאמא ✅

> כל פעם שתעשו `git push` עם שינוי — Render יעדכן את האתר אוטומטית.

## 🔧 איך זה עובד מבפנים

- התצוגה המקדימה משקפת את מה שייצא ל‑PDF.
- בלחיצה על הכפתור נוצר קובץ PDF **ישירות בקוד** (בדפדפן) באמצעות **pdf-lib**:
  הלוגו מוטמע כתמונה, והטקסט מצויר **תו‑אחר‑תו במיקום מוחלט**, לפי סדר ויזואלי
  שמחושב על‑ידי מנוע **bidi** (כך עברית ומספרים מעורבים יוצאים נכון תמיד, בכל מציג).
- הגופן (Alef) מוטמע בקובץ, יש **חלוקה לעמודים** אוטומטית, והקובץ **יורד / משותף ישירות**
  (בלי חלון הדפסה ובלי שכתובת האתר תופיע במסמך).
- **למה לא html2canvas / הדפסה?** `html2canvas` שיבש מספרים בעברית (RTL/bidi) ולעיתים
  הוציא עמוד ריק בטלפון; הדפסת הדפדפן הוסיפה כותרת/כתובת אתר ולא תמיד נשלטה.
  יצירת ה‑PDF ישירות בקוד פותרת את כל אלה ונותנת פלט זהה בכל מכשיר.
